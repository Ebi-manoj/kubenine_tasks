

### What is Containers

A container is a lightweight, isolated runtime environment that packages an application along with its dependencies, while sharing the host system’s OS kernel.

### what problem of VM solves by container

Virtual machines solve isolation by running a full operating system for each instance, but this introduces significant overhead in terms of memory, CPU, and startup time. 

Containers solve this problem by sharing the host OS kernel and packaging only the application and its dependencies, making them lightweight, faster to start, and more resource-efficient while still providing isolation using namespaces and cgroups.


### How does Buildah works

Docker relies on a central daemon (dockerd), which can become a control-plane bottleneck or failure point, whereas Buildah uses a daemonless approach to build OCI images directly using kernel features, improving reliability and security.

### Docker Architecture

Docker follows a client-server architecture.

The Docker CLI acts as the client, allowing users to run commands like `docker run` or `docker build`. These commands are sent to the Docker daemon (dockerd) through a REST API, typically over a Unix socket.

The Docker daemon is responsible for managing images, containers, networks, and volumes. Internally, it uses containerd to manage container lifecycle and image operations, and runc as the low-level runtime to create and run containers using Linux kernel features like namespaces and cgroups.

In short, the CLI sends requests, the daemon processes them, and the actual container execution is handled by containerd and runc.
