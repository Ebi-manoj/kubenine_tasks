
### 1. What is the difference between a Docker image and a Docker container?

Docker image is a read-only template ,we can consider it as a blueprint for the containers.
Docker container are actual running instance made from the image with its own isolated runtime.

### 2. What command did the busybox container execute?

busybox executed echo commands as main process and print "Hello world"
### 3. Why did the container stop after the command finished?

Container only stay alive as long as its main process is running. In our case main process (echo)
executed immediately and finishes, so the container also exited.
### 4. Why does the container not appear in docker ps but appear in docker ps -a?

docker ps only includes the running container , but docker ps -a includes all container even if it is stopped or exited
### 5. Why is a container exit not always an error?

Container can exit normally when its main process finish its execution and its not an error.
### 6. What is the purpose of docker logs?

docker logs shows all the output of (stdout and stderr ) the container which is stored in the docker host directory. (/var/lib/docker/container/container_id/json)
### 7. What is the purpose of docker rm?

It is used to remove the stopped/exited containers.
### 8. Why can multiple containers be created from the same image?

Docker image is only a template and Docker can create multiple independent container instances from it, each with its own runtime state.
### 9. What is the main process idea in a container?

Process inside the container having PID 1 is called a main process , when its execution finishes container will exit normally.

It helps you understand container lifecycle, logs, and process behavior, which are essential when diagnosing why a container stopped or isn’t behaving as expected in real systems.
