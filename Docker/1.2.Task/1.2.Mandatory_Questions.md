
### 1. Why did the NGINX container keep running while the busybox container exited?

NGINX runs a long-running web server process that keeps listening for requests, so the container stays alive. BusyBox ran a short command (echo) which finished immediately, so the container exited.

### 2. What does detached mode mean in Docker?

Detached mode (-d) means the container runs in the background and does not block your terminal.

### 3. What does -p 8080:80 mean exactly?

It maps port 8080 on the host to port 80 inside the container, allowing external access to the container’s service.

### 4. What is the difference between host port and container port?

The host port is on your machine, while the container port is inside the container where the application is actually running

### 5. How did traffic from the browser reach the container?

The browser request reaches the container through the host network stack, where Docker uses iptables NAT rules to forward traffic from the host port to the container’s IP and port via the docker bridge network.

### 6. What is the purpose of docker logs?

It shows the output (stdout and stderr) of the container, which helps in debugging and understanding what the container is doing.

### 7. What is the purpose of docker inspect?

It provides detailed information about the container’s configuration, state, networking, and other metadata.

### 8. What is the purpose of docker top?

It shows the running processes inside the container from the host’s perspective.

### 9. What is the purpose of docker exec?

It allows you to run commands or open a shell inside a running container for debugging or inspection.

### 10. What is the purpose of docker stats?

It shows real-time CPU, memory, and network usage of containers.
   
### 11. What did you observe about CPU and memory usage for the container?

The NGINX container used very low CPU and memory because it was idle and only serving minimal requests

### 12. Why is checking resource usage useful in production?

It helps detect performance issues, resource bottlenecks, and prevents containers from overusing system resources.

### 13. What happened when you ran docker stop?

Docker sent a signal to stop the main process (NGINX), and once it stopped, the container exited.

### 14. Why does removing the container not remove the image?

Because the image is a reusable template, and containers are just instances created from it

It teaches how containers run, how to access them, check logs, inspect state, and monitor resources, which are essential for diagnosing issues in real systems.
