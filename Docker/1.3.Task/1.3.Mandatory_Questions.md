

### 1. What is a Dockerfile?

Dockerfile is a set of instruction used by the docker for creating the application image ,with its base image , dependencies , file systems and command for executing when container starts.

### 2. Why is a Dockerfile required when packaging an application into an image?

A Dockerfile is used to automate building an image, reduce manual effort, and ensure the application runs the same way everywhere.

### 3. What does docker build do?

Docker build will create a image with image-tag we specified (-t) using the set of instructions we wrote in the Dockerfile
### 4. What does docker run do?

Docker will first check the specified image locally, if its not found it will pull from registry .Then docker will create a writable layer on the top of the image layer, set up environment needed for isolation, after that CMD command will run as the main process of container.
### 5. What is the purpose of requirements.txt in this task?

requirements.txt is used to provide dependencies need for our flask-app to run. so we dont have  write each installation command of dependencies manually. we can simply do PIP install requirements, it will install all the dependencies.

### 6. Why must the Flask app bind to 0.0.0.0 inside the container?

By default, flask app is only binds to 127.0.0.1, so our app is only listening in the container only. Binding to `0.0.0.0` allows the app to listen on all network interfaces, so it can accept requests from outside the container.
### 7. What does -p 5000:5000 mean?

we are doing a port mapping here , mapping the host port 5000 to container port 5000
### 8. What is the purpose of WORKDIR in the Dockerfile?

Specify the working directory of the container, then all the command will run from this directory.
### 9. What is the difference between RUN and CMD in the Dockerfile?

Run command is used to run a instruction or command when the docker is building the image. Run can be used multiple times inside a docker file.
CMD will execute only when the container started and it will execute as the container main process
### 10. Which project files are included in the Docker image?

app.py and requirements.txt

### 11. What is the purpose of docker logs in this task?

Dockers logs helps us to show the output of container (stdout and stderr).In this task we can see the logs of HTTP request, it is useful for debugging what happened for a specific requests 
### 12. What is the purpose of docker inspect in this task?

docker inspect provides detailed metadata about the container, including configuration, state, network settings, and port mappings.
### 13. What is the purpose of docker exec in this task?

docker exec is used to run commands inside a running container, which helps in inspecting files or debugging.
### 14. Why did the Flask container stay running?

The Flask app (python app.py) is a long-running process that continuously listens for HTTP requests. Since the main process is running, the container stays alive.
### 15. Why does removing the container not remove the image?

Containers and images are separate. An image can be used to create multiple containers, so deleting a container does not remove the image.
