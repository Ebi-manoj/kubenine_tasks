
#### What I find in debugging task 1.4

The Dockerfile did not produce any runtime error because the Flask application only required Flask, which was installed directly using `pip install flask`.

However, the configuration is incorrect from a packaging and reproducibility standpoint:

1. requirements.txt is not copied or used → dependencies are not reproducible
2. Only app.py is copied → incomplete application packaging
3. Dependency versions are not locked → risk of version drift
4. Not following Docker layer caching best practices
5. Not scalable for real-world applications

So, while the container works now, this Dockerfile is not reliable, maintainable, or production-ready.
