## Docker Storage & Persistence

### Why storage is needed

- Containers are **ephemeral**
- Data inside container layer is **lost when container is removed**
- Need external storage → **persistent data**

---

## Docker Volume (Concept)

- A **Docker volume** is persistent storage managed by Docker
- Stored on host (usually `/var/lib/docker/volumes/`)
- Independent of container lifecycle
- Survives container deletion

Container (/app/data)  
        ↓  
Docker Volume (mysql-data)  
        ↓  
Host (/var/lib/docker/volumes/mysql-data/_data)

---

## Types of Docker Storage

### 1. Named Volume (Recommended)

docker volume create mydata  
docker run -v mydata:/app/data image

**Characteristics**

- Managed by Docker
- Easy to use
- Portable
- Best for production (with orchestration)

**Inspect**

docker volume inspect mydata

---

### 2. Bind Mount

docker run -v /home/ebi/data:/app/data image

**Characteristics**

- Direct mapping to host path
- Full control over files
- Useful for development

**Drawbacks**

- Depends on host structure
- Less portable

---

### 3. Anonymous Volume

docker run -v /app/data image

**Characteristics**

- No explicit name
- Docker auto-generates ID
- Hard to track/manage
