
### 1. What is a Docker bridge network?

As we know containers are isolated within its own network namespace, so if we need to communicate to the container we need a network interface. Docker solves this issue by creating a Docker bridge , which is virtual ethernet (veth0) also known as docker0 . Container eth0 is connected to this bridge and host etho0 is connected to this bridge. This ensure communication.
### 2. Why was a custom Docker network created in this task?

We have a mysql container which should be isolated from external and also need a php container to access this container . So we put those in a custom network so that both can communicate properly using service to service discovery without using hardcoded IP. Ips are reliable and it is ephemeral.
### 3. Why does `localhost` not work for container-to-container communication?

localhost inside the container is only pointing to container itself so the communication is not possible outside the container.
### 4. Why does using the container name `mysql-net-demo` work?

We are in a custom network , so internally docker will create a DNS server typically 127.0.0.11 which resolve container name to its Ip address in that scope. We can see it also in the docker inspect DNS names are given and it is equal to container name.
### 5. How does Docker resolve one container name to another container on the same network?

As i said before docker uses a internal DNS server to resolve container name container IPs .
### 6. Why is using container names better than using hardcoded IP addresses?

Because IP are ephemeral , when container restart or create again it will be different . So it is not reliable to use IP address this case as service to service communication can break. But using container name it will resolve exactly to the same IP of the container even it is restarted or not. Because docker will dynamically update these values when it restart.
### 7. What did `docker network inspect app-net` show you?

Docker network inspect app-net shows its network configuration such as subnet level , IP in use , more importantly what all container are attached to this network.
### 8. What did `docker inspect mysql-net-demo` show about networking?

It shows the networking configuration such as which network it belongs to , IP address of the container , DNS names , mac address , network name (in our case app-net).
### 9. Why was PhpMyAdmin exposed to the host, but MySQL was not?

Because it was a UI and it should be access from the browser and it should exposed to host. Then only request will reach the container.
In the case of MySQL it should not expose publicly because it might have sensitive data and it should kept in isolated network.
